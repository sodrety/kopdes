# Use professional cooperative back-office UI

Accepted. Supersedes ADR 0003: KKSUK PD Dharma Jaya should use a professional cooperative back-office visual system rather than the Wise-inspired friendly fintech style. Keep ADR 0008's Bahasa-first operational conventions, light cooperative palette, and domain/status accents, but use restrained typography, denser spacing, neutral shell surfaces, 8px maximum radius for ordinary UI, pill shapes only for status badges, and table-first admin screens where charts and KPI tiles are secondary summaries.

**Consequences**

- `DESIGN-wise.md` is historical inspiration only, not the source of truth for implementation.
- Admin screens should prioritize operational density, tables, filters, and review controls over card-heavy presentation.
- Member screens should share the same professional foundation while allowing slightly warmer spacing.
